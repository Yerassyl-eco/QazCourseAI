## Цель
Перенести содержимое архива `qazcourse-ai-github.zip` в текущий пустой проект Lovable, чтобы код синхронизировался с созданным GitHub-репозиторием.

## Анализ архива
- Стек: TanStack Start + React 19 + Tailwind CSS v4 + shadcn/ui — полностью совместим с текущим проектом
- Приложение: QazCourse AI — платформа для поиска и рекомендации курсов в Казахстане
- Есть Supabase-интеграция, i18n, AI-ассистент, система рекомендаций
- В архиве есть `.git` и `.workspace/.git` — их удалим при переносе

## Шаги реализации

### 1. Очистка и подготовка
- Удалить placeholder-контент из текущего проекта (`src/routes/index.tsx`)
- Удалить директории `src/components/`, `src/hooks/`, `src/lib/`, `src/routes/`, `src/integrations/` — заменим содержимым архива
- Сохранить конфиги Lovable: `.lovable/`, `.workspace/`, `.git/`, `node_modules/`, `bun.lock`

### 2. Копирование файлов из архива
- `src/` — полностью заменить (компоненты, хуки, lib, маршруты, интеграции)
- `package.json` — смержить зависимости, оставить `bun` в качестве пакетного менеджера
- `vite.config.ts`, `tsconfig.json`, `src/styles.css` — заменить
- `supabase/` — скопировать конфигурацию
- `.env.example`, `SECRETS.md` — скопировать как справочные
- Исключить: `.git/`, `.workspace/.git/`, `package-lock.json`

### 3. Установка зависимостей
- Установить все зависимости из архивного package.json
- Убедиться, что `@lovable.dev/vite-tanstack-config` и `@lovable.dev/cloud-auth-js` присутствуют

### 4. Верификация
- Проверить `bun run build:dev` на ошибки
- Убедиться, что все импорты разрешаются
- Проверить, что маршруты регистрируются корректно

## Риски и решения
- **Конфликт bun vs npm**: текущий проект использует bun, архив — package-lock.json. Решение: использовать bun, удалить package-lock.json.
- **Git-конфликты**: архив содержит `.git`. Решение: не копировать `.git` и `.workspace/.git`.
- **Секреты/env**: архив содержит `.env.example`. Решение: не копировать .env, использовать SECRETS.md как справку.
